# 备份控制域

本上下文描述从受保护数据源创建、管理并恢复版本化备份所使用的统一语言。

## Language

**版本化备份**：
按时间点保留源数据的可恢复历史版本；目标端不要求呈现为与源端一致的普通文件目录。
_Avoid_：同步、镜像同步、目录复制

**备份节点**：
安装并运行本应用的设备；它拥有或本地暴露待保护数据，并负责创建快照及发起恢复。
_Avoid_：同步端、客户端

**管理员**：
获准登录并完全管理一个应用实例的单一人员身份。
_Avoid_：普通用户、操作员、多租户

**控制服务**：
在备份节点上常驻运行并负责管理页面、调度及备份控制的应用进程；其生命周期独立于浏览器会话。
_Avoid_：桌面窗口、管理页面

**服务账号**：
操作系统中运行控制服务及其备份执行引擎的身份；它界定应用可读取的源目录范围。
_Avoid_：应用管理员、远程 SSH 用户

**管理页面**：
由控制服务提供、供管理员在浏览器中操作应用的 React 界面；关闭页面不会停止控制服务或任务运行。
_Avoid_：桌面应用、控制服务

**兼容性中心**：
集中展示备份节点环境检测结果、受影响功能和修复路径的管理页面区域。
_Avoid_：启动失败页、普通日志

**告警**：
要求管理员关注的运行或仓库状态变化，例如部分成功、最终失败、长期无完整成功或仓库异常。
_Avoid_：普通日志、成功记录

**通知渠道**：
把告警投递到应用之外的可替换出口；首版实现 ntfy。
_Avoid_：应用内通知、日志输出

**备份源**：
由一个备份任务读取并形成快照的数据对象；首版可以是源目录，也可以是由 MySQL 或 PostgreSQL 连接器产生的逻辑数据库导出流。
_Avoid_：备份目录、Restic 路径

**源目录**：
备份节点上被选中纳入版本化备份的一种文件系统备份源。
_Avoid_：备份目录（该词无法区分数据来源与仓库存放位置）

**数据库源**：
可由备份节点访问、并在任务运行时通过受控连接器生成逻辑导出流的一个 MySQL 或 PostgreSQL 逻辑数据库。
_Avoid_：数据库文件、数据目录（直接复制运行中数据库的数据文件不能替代逻辑导出）

**数据库实例**：
一个运行中的 MySQL 或 PostgreSQL 服务，包含一个或多个逻辑数据库以及可能跨数据库生效的账号、角色、权限或表空间等全局对象。
_Avoid_：数据库源、数据库备份任务

**数据库连接配置**：
由管理员命名并复用的、连接一个数据库实例的持久配置，包含数据库类型、网络端点、传输安全设置、已验证工具链和秘密库中的凭据；它可以承担备份或恢复其中一种用途。
_Avoid_：数据库实例、数据库备份任务、临时命令

**备份数据库凭据**：
数据库连接配置中只用于读取源数据库并执行逻辑导出的账号与秘密；它不应具有创建或修改目标数据库所需的权限。
_Avoid_：数据库恢复凭据、应用管理员

**数据库连接器**：
封装一种数据库的兼容性检测、逻辑导出及逻辑导入行为的深模块；首版实现 MySQL 与 PostgreSQL 两个适配器。
_Avoid_：Shell 脚本、任意命令、Restic 后端

**数据库客户端工具链**：
由数据库连接器使用的一组已验证官方客户端程序；首版为 MySQL 的 `mysqldump` 与 `mysql`，以及 PostgreSQL 的 `pg_dump` 与 `pg_restore`。它不同于数据库服务端和 Restic。
_Avoid_：数据库实例、应用托管的 Restic

**工具链探测**：
在任务启用及实际运行前确认数据库客户端工具链存在、版本和能力符合目标数据库实例要求的检查；未通过时数据库任务只能保持草稿状态。
_Avoid_：仅检查 PATH、数据库连接测试

**数据库一致性责任**：
为使逻辑导出代表可恢复业务状态而由数据库管理员承担的源端职责，包括表引擎、事务语义、锁定策略和维护窗口；首版应用不代替管理员检查或强制这些条件。
_Avoid_：Restic 完整性检查、仓库维护

**逻辑数据库导出**：
数据库官方客户端根据结构化任务配置生成、可由对应导入工具恢复的数据流；它包含单库的结构和数据，但不承担复制数据库实例账号或密码。该数据流默认直接进入 Restic，不在备份节点持久保存明文中间文件。
_Avoid_：数据库数据目录副本、用户脚本

**数据库恢复凭据**：
恢复时用于连接目标数据库实例并创建或写入目标数据库的账号与秘密；它由管理员输入或从秘密库选择，不等于源实例中原有的应用账号。
_Avoid_：仓库密码、被备份的数据库账号

**数据库恢复目标**：
管理员在恢复时选定的目标数据库实例及逻辑数据库；首版只允许新数据库或经预检确认没有用户对象的空数据库。
_Avoid_：恢复目标目录、正在使用的非空数据库

**空数据库**：
只含数据库引擎必要的系统对象和默认结构、尚无用户创建的表、视图、例程、事件或其他业务对象的逻辑数据库。
_Avoid_：业务已停写的数据库、可以覆盖的数据库

**数据库快照元数据**：
随数据库快照保存、用于解释和恢复逻辑导出的非秘密信息，包括数据库类型与版本、导出格式和工具版本、字符集、编码及排序规则等创建参数。
_Avoid_：数据库凭据、仅存在应用 SQLite 中的运行摘要

**实例身份对象**：
数据库实例范围内的账号、角色、密码、全局授权和 PostgreSQL 表空间等对象；首版单库备份不负责保护或自动恢复它们。
_Avoid_：数据库恢复凭据、单库结构与数据

**排除规则**：
隶属于单个备份任务、明确使匹配内容不进入快照的已启用规则。
_Avoid_：忽略文件、隐藏默认值

**排除建议**：
应用根据源目录内容提出、但尚未生效的候选排除规则；只有管理员确认后才转为排除规则。
_Avoid_：自动排除

**备份任务**：
将一个备份源保护到一个独立备份仓库的可单独执行单元；首版任务类型包括目录备份和数据库逻辑备份。
_Avoid_：同步任务、目录映射

**数据库备份任务**：
把数据库实例中的一个逻辑数据库导出到一个独立备份仓库的备份任务；同一实例中的多个数据库必须建立多个任务，但可以由同一计划统一触发。
_Avoid_：整实例备份、数据库列表任务

**任务草稿**：
已保存但因配置未完成或前置检测未通过而不能运行、也不能加入有效计划的备份任务。
_Avoid_：失败任务、已禁用任务

**任务运行**：
备份任务的一次端到端执行过程，可由时间表或管理员手动触发，并以成功、部分成功、失败、取消或跳过结束。
_Avoid_：备份任务（任务是持久配置，运行是一次端到端过程）

**运行摘要**：
任务运行的结构化结果、持续时间、数据量和错误计数记录，默认长期保留。
_Avoid_：原始运行日志、快照

**原始运行日志**：
经秘密脱敏后的 Restic 详细输出，用于短期诊断并受时间与磁盘上限控制。
_Avoid_：运行摘要、审计记录

**审计记录**：
管理员登录、安全、配置和破坏性操作的不可逐条删改事件记录，用于追溯谁在何时改变了什么。
_Avoid_：运行摘要、原始运行日志

**执行尝试**：
任务运行内部对 Restic 的一次实际调用；一次运行可因可重试故障包含多次执行尝试。
_Avoid_：任务运行、计划触发

**部分成功**：
目录备份任务运行创建了快照，但一个或多个应备份的源文件未能读取；该运行不更新最近一次完整成功时间。数据库导出工具非零退出时不创建快照，而是失败。
_Avoid_：成功、失败、普通警告

**任务定时执行**：
隶属于单个备份任务、在指定时间触发该任务的持久安排。管理页面在任务详情中配置它；底层可继续使用计划记录保存调度状态和历史发生。
_Avoid_：独立备份计划模块、Cron、定时脚本

**计划时间表**：
描述任务定时执行何时触发的结构化规则，例如每日、每周或固定间隔。
_Avoid_：Cron、定时脚本

**资源策略**：
限制任务运行所用带宽、文件读取并发、压缩强度、CPU 和全局并行数量的配置。
_Avoid_：保留策略、Restic 任意参数

**快照**：
一次成功备份所形成的、代表特定时间点源数据状态的不可变历史视图。
_Avoid_：同步结果、副本

**保留策略**：
隶属于单个 Restic 仓库、用于决定哪些历史快照继续保留的规则集合。任务详情可提供指向所选仓库策略的快捷编辑入口，但不保存第二份策略。
_Avoid_：计划策略、全局清理规则

**保留窗口**：
从任务仓库的最新成功快照向过去计算、其内快照应继续保留的一段时间。
_Avoid_：从维护当天计算的文件存活天数

**仓库维护**：
按照仓库的保留策略移除过期快照并回收其空间的排他性操作。
_Avoid_：备份运行、目录清理

**完整性检查**：
在不改变快照内容的前提下验证备份仓库结构，并可分批或完整读取仓库数据以发现损坏的操作。
_Avoid_：恢复测试、仓库修复

**恢复操作**：
由管理员发起、把选定快照中的全部或部分内容写入恢复目标目录，或通过数据库连接器导入目标数据库的过程。
_Avoid_：反向同步、备份任务

**恢复目标目录**：
恢复操作在备份节点上写入所选内容的目录。
_Avoid_：源目录（两者可能相同，但角色不同）

**备份仓库**：
保存快照及其数据、并可独立接入应用的目标集合；它可以暂时不绑定备份任务，其中的数据通过备份系统读取和恢复，而不是作为源目录的可直接浏览镜像。
_Avoid_：同步目录、镜像目录

**异常仓库**：
最近一次完整性检查发现损坏、因而暂停全部自动写入与自动修复并等待管理员处置的备份仓库。
_Avoid_：离线仓库、连接失败

**仓库密码**：
用于解锁备份仓库加密数据的用户可知秘密；它不同于建立 SFTP 连接所使用的 SSH 私钥。
_Avoid_：密钥（无法区分仓库密码与 SSH 私钥）、SSH 密码

**秘密库**：
应用用于加密托管 SSH 私钥、仓库密码及其他可解密秘密的受保护存储。
_Avoid_：数据库、密码表

**秘密库口令**：
仅在重启后锁定模式中由管理员提供、用于解锁秘密库的独立口令。
_Avoid_：管理员登录密码、仓库密码

**远程仓库**：
位于备份节点之外、通过网络访问的备份仓库。
_Avoid_：远端同步目录、远程副本

**远程主机**：
通过 SFTP/SSH 向备份节点提供一个或多个远程仓库的设备；其连接身份和受信主机指纹可被这些仓库复用。
_Avoid_：远程仓库、备份节点
