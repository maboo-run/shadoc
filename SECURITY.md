# 安全政策

## 支持版本

在 `0.x` 阶段，仅最新稳定版本接收安全修复。安全修复发布后，旧版本可能不再继续维护；请关注 GitHub Releases 并及时升级。

## 私密报告漏洞

请使用仓库的 [GitHub Private Vulnerability Reporting](https://github.com/maboo-run/shadoc/security/advisories/new) 提交漏洞。不要通过公开 Issue、Discussion 或未经脱敏的日志报告安全问题。

报告应尽量包含：

- 受影响版本和平台；
- 攻击前提、影响和可复现步骤；
- 涉及的入口、资源和安全边界；
- 已经做过的验证；
- 不包含真实凭据或第三方用户数据的最小化证明材料。

维护者会先确认报告是否可复现和是否属于安全问题，再通过私密 Advisory 协调修复与披露。修复可用前，请不要公开漏洞细节。

## 敏感信息

报告中不得包含真实仓库密码、SSH 私钥、数据库凭据、访问令牌、Agent 证书、内部主机清单或未经脱敏的完整诊断包。如果必须证明秘密泄漏，请使用专门创建且已经撤销的测试凭据。
